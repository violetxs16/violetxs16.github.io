NutriHall

React, Node.js, Firebase, Jest

NutriHall is an AI-powered dining hall assistant designed to help UCSC students make healthier eating choices in a simple and efficient way. It allows users to select their dietary preferences and health goals, and provides meal options from campus dining halls that align with these choices. Once preferences are entered, NutriHall generates a personalized meal plan that includes nutritional breakdowns and portion size recommendations. It also tracks dietary goals over time while aiming to reduce food waste.

Slug Dining

C, Makefile, Docker

This project involves implementing a multithreaded dining hall management system in C that uses POSIX threads for efficiency and synchronization mechanisms to control access to the dining hall. The system needs to handle students entering and leaving the dining hall while ensuring only a certain number of students can be inside at a time. Additionally, it must manage a cleaning service that requires the dining hall to be empty before it can begin cleaning.

Web Crawler

C

The Web Crawler, is a standalone program that crawls the web and retrieves web pages starting from a “seed” URL. It parses the seed webpage, extracts any embedded URLs, then retrieves each of those pages, recursively, but limiting its exploration to a given “depth”.

Nano Interpreter

Haskell

The Nano Interpreter project involves implementing a functional programming language interpreter for Nano in Haskell, using abstract syntax trees and environments to evaluate programs. The interpreter supports integers, Booleans, lists, arithmetic and logical operations, conditionals, let-expressions, and recursive functions. It also handles list operations like head and tail, ensuring proper scoping, binding, and error handling.

  • NI Code

LRU Cache

C

This project implements an LRU (Least Recently Used) Cache in C, designed as an abstract data type with a fixed capacity of three items. When the cache reaches its limit, the least recently used item is automatically evicted to make space for new entries. The implementation combines a set and a hashtable ADT to enable efficient operations, including insertion, lookup, iteration, deletion, and printing of cache contents.
